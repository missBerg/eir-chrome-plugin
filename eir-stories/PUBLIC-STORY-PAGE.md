# Eir Stories Public Story Page

## Objective

The public story page should be readable by someone who is stressed, tired, and searching for relevance quickly.

It should feel like:
- a thoughtful case history
- a patient narrative
- a structured timeline

It should not feel like:
- a social feed post
- a chart-heavy dashboard
- a medical records dump

## Page Structure

### 1. Story Header

Contains:
- story title
- author display name or pseudonym
- short summary
- evidence level
- key topic tags

### 2. Why This Story May Be Relevant

A short machine-assisted or hand-authored summary:
- same condition
- same symptom pattern
- same treatment path

### 3. Journey Narrative

The core story in human language.

### 4. Timeline

Chronological events with:
- date precision
- event type
- short description
- optional evidence badge

### 5. Health Context

Optional structured section:
- conditions
- symptoms
- medications
- procedures
- outcomes

### 6. What Helped / What Did Not

Short practical reflection section.

### 7. Related Stories

Surface adjacent journeys without turning the page into a recommendation feed.

## Metadata To Show

Publicly visible:
- evidence level
- country
- broad care context
- condition tags
- symptom tags

Use caution with:
- exact dates
- exact institutions
- exact locations

## Reading Experience

Priorities:
- strong typography
- generous spacing
- low-noise layout
- mobile-first readability

## Interaction Model

At MVP, keep actions limited:
- save
- share
- report
- follow updates later

Avoid public comments in the first version unless moderation is already robust.
